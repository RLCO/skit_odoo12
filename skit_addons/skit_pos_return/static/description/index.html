POS return

This application helps you to return product using barcode scan.

Watch the Video
Key features:
  • A barcode is generated while validating an order.The product can be returned by scanning the Order's barcode(Receipt Code).
  • Return validity days is given for individual Products and its Category.
  • POS user can return order partially or fully by "Complete Return" button.
  • Returned order will not allow to return again.
  • An order/ Product can be returned only if the invoice is in paid state.(Will not allow "Open" state products).
  • Cancel an order (inappropriate product or Qty) with confirmation message. It will void payment and invoice journal entries.

Returnable Condition

For returning a product, check the "Returnable" checkbox in the products.

Now the return days condition can be given in "Products"(Field - 'Days of return') and "Categories"(field - 'Return Days').

Return an Order

Place an order in POS. The order generates a "Barcode" in the receipt.

The product can be returned by "Scanning".

Select the code and Return

The product can also be returned using the "Return Button"(in POS) by giving the Order Receipt Code(Available in POS Orders).

Error Pop-ups

If the return quantity is greater than available quantity, an error message occurs in the footer.

And if the product is "Open" state, the product cannot be returned

For Returning the Damaged products

Now the order can be returned and validated.

And the Return Order Line will be created in the POS orders.

Backend Process

After validating the products, the order will be created in the POS order screen.

Select the created return order and the return line will be available based on the selected quantity.
